# Quickstart: Mobile Drawing Party Game

**Feature**: `001-draw-guess-mobile` | **Date**: 2026-07-23

## Prerequisites

- Node.js 20
- Copy `.env.example` → `.env` and set `HOST_USERNAME`, `HOST_PASSWORD_HASH`
  (bcrypt hash), and `JWT_SECRET`. See `DEPLOYMENT.md` for how to generate a
  password hash. These are unchanged by this feature.

## Install & run

```bash
npm install
npm run dev            # starts backend (Fastify, tsx watch) + frontend (Vite) together
```

- Backend: `http://localhost:3000` (or `PORT` from `.env`)
- Frontend dev server proxies to the backend for `/api/*` and `/ws` (see
  `frontend/vite.config.ts`)

## Manual golden-path check for this feature

1. Open the app on a desktop browser, log in with the host credentials
   (`/#/login`).
2. Create a session with an initial phrase; confirm the QR code and join URL
   render in French copy.
3. On a second device (or a new browser profile, to simulate a separate
   client), open the join URL, confirm the nickname form appears (session is
   `lobby`), submit a nickname, confirm the "en attente" waiting message
   appears with **no** visible/interactive drawing canvas yet.
4. Join a second player using the **same nickname** as the first — confirm
   both appear as-is in the host's live roster (no " 2" suffix).
5. On the host, click "Commencer la partie" — confirm both player screens
   transition to the game view showing the phrase label and an interactive
   drawing area filling the remaining space; confirm freehand strokes render
   immediately on touch/pointer input.
6. On the host, type and publish a new phrase — confirm both player screens
   update without any player action. Try submitting an empty phrase — confirm
   it is rejected and the previous phrase stays active.
7. Force-quit and relaunch the app (or fully close/reopen the browser tab) on
   one player's device — confirm it resumes directly to the game screen
   without re-asking for a nickname.
8. On the host, click "Terminer la partie" — confirm both player screens move
   to a closing screen, and that a fresh join attempt at the same URL now
   shows "Cette partie est terminée" instead of the nickname form.
9. Open a join URL for a session ID that was never created — confirm it shows
   a distinct "Partie introuvable" message.
